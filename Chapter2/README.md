# 第二章代码说明

本目录对应第二章的配套代码，围绕“分子性质预测—不确定性评估—可解释性分析”三个部分展开，使用 **PyTorch Geometric (PyG)** 与 **RDKit** 完成一个较完整的教学示例。代码以 Jupyter Notebook 形式组织，适合课堂演示、实验教学和后续二次开发。

---

## 一、文件结构

- `2.1 分子性质预测.ipynb`  
  第二章中“分子性质预测”部分的主 Notebook，演示如何基于 GIN 完成分子分类与回归任务。

- `2.2 不确定性评估.ipynb`  
  第二章中“不确定性评估”部分的 Notebook，演示 **MC-Dropout** 与 **EDL（Evidential Deep Learning）** 在分子性质预测中的应用。

- `2.3 可解释性分析.ipynb`  
  第二章中“可解释性分析”部分的 Notebook，演示 **Integrated Gradients (IG)** 与 **Substructure-Mask Explanation (SME)** 在分子性质预测中的解释过程。

---

## 二、代码内容概览

### 1. `2.1 分子性质预测.ipynb`

该 Notebook 用于介绍分子图神经网络在基础性质预测任务中的典型流程，主要包括以下内容：

1. 导入必要依赖，包括 `torch`、`torch_geometric`、`rdkit`、`numpy` 和 `pandas` 等。
2. 载入分子性质预测数据集，示例任务包括：
   - **BBBP**：二分类任务，用于血脑屏障渗透性预测；
   - **ESOL**：回归任务，用于水溶解度预测。
3. 展示分子图数据的基本结构，包括：
   - 节点特征 `x`
   - 边索引 `edge_index`
   - 边特征 `edge_attr`
   - 分子可视化与原子编号显示
4. 构建通用的 **GIN** 主干网络，并分别实现：
   - `GINClassifier`：分类模型；
   - `GINRegressor`：回归模型；
   - `GINMaskRegressor`：为后续 mask 类解释方法保留接口。
5. 给出训练、验证和测试的完整流程，包括数据划分、单轮训练、分类评估与回归评估。
6. 保存训练后的模型参数与预测结果，便于后续不确定性分析与可解释性分析直接复用。
7. 提供若干探究题，引导读者进一步考察 GIN 的层数、Dropout 和隐藏维度等超参数对性能的影响。

该 Notebook 的默认超参数较为统一，例如隐藏维度为 128、层数为 4、Dropout 为 0.2、学习率为 `1e-3`、训练轮数为 100，可作为第二章后续代码的基础配置。

### 2. `2.2 不确定性评估.ipynb`

该 Notebook 在分子性质预测模型的基础上，进一步演示如何估计预测结果的不确定性，主要包括以下内容：

1. 重新训练或加载 GIN 分类器与回归器，作为不确定性分析的基础模型。
2. 使用 **MC-Dropout** 进行测试时采样：
   - 在推理阶段仅打开 Dropout；
   - 对同一分子进行多次前向传播；
   - 对分类任务统计平均概率与方差；
   - 对回归任务统计预测均值与标准差。
3. 按“高置信度/低置信度”样本进行排序和观察，帮助读者理解不确定性与预测难度之间的关系。
4. 演示 **EDL** 在分子性质预测中的应用，包括：
   - 分类任务中的 **Dirichlet evidential** 建模；
   - 回归任务中的 **Normal-Inverse-Gamma (NIG)** 建模。
5. 给出实验探究问题，例如：
   - 高不确定性样本是否更容易预测错误；
   - 低不确定性是否一定意味着预测正确；
   - MC 采样次数增加是否一定带来更好的估计效果。

该 Notebook 的目的不是只给出一个“分数”，而是让读者看到：模型除了输出预测值外，还可以进一步输出与之配套的**置信信息**，从而为实际应用中的风险判断提供依据。

### 3. `2.3 可解释性分析.ipynb`

该 Notebook 主要面向分子性质预测中的可解释性分析，以 **ESOL** 回归任务为例，演示模型为何给出某一预测结果。主要内容包括：

1. 训练或加载回归模型，作为后续解释分析的基础。
2. 使用 **Integrated Gradients (IG)** 对输入原子特征进行归因，得到每个原子对预测结果的重要性评分。
3. 使用 **SME（Substructure-Mask Explanation）** 进行子结构级解释：
   - 基于分子片段或子结构进行 mask；
   - 比较 mask 前后的预测变化；
   - 判断哪些子结构对模型预测最关键。
4. 借助 RDKit 可视化解释结果，将原子或子结构的重要性映射到分子结构图上。
5. 设置若干讨论问题，引导读者思考：
   - 模型高亮的位置是否符合化学直觉；
   - 可解释性方法的价值更偏向“证明模型正确”，还是“帮助研究者提出假设”；
   - 随着模型变复杂，解释结果是否也会更难理解。

该 Notebook 的重点在于帮助读者理解：可解释性分析并非简单地给出“标准答案”，而是为分子设计和性质分析提供一个可视化的、可讨论的证据入口。

## 三、建议运行顺序

建议按照以下顺序运行：

1. **`2.1 分子性质预测.ipynb`**  
   先完成基础模型训练，并熟悉分子图数据结构。

2. **`2.2 不确定性评估.ipynb`**  
   在基础预测模型之上，进一步分析模型预测的可信度。

3. **`2.3 可解释性分析.ipynb`**  
   在得到可用模型后，对具体分子的预测结果进行解释。

虽然三个 Notebook 都可以相对独立运行，但按上述顺序执行更符合第二章的教学逻辑，也便于复用前面保存的模型参数。

---

## 四、运行环境建议

建议使用如下 Python 环境：

- PyTorch
- PyTorch Geometric
- RDKit
- NumPy
- Pandas
- Jupyter Notebook / JupyterLab
- IPython（用于 Notebook 内可视化显示）

如果本地尚未安装 PyG 或 RDKit，建议优先根据当前系统和 CUDA 版本完成对应依赖配置，再运行 Notebook。

---

## 五、运行后可能生成的结果

在运行过程中，代码通常会自动创建如下目录或文件：

- `./data/`：用于下载和缓存 MoleculeNet 数据集；
- `./saved_models/`：用于保存训练好的模型参数；
- 预测结果文件（如验证集或测试集的 `.csv` 文件）；
- 分子解释图、归因图或相关可视化输出。

如果需要重复实验，建议保留 `saved_models` 目录，避免每次都重新训练全部模型。

---

## 六、教学使用建议

这组代码适合配合第二章内容用于以下场景：

1. **课堂演示**：帮助学生直观理解分子图表示、GIN 模型和性质预测流程；
2. **实验课练习**：让学生自行修改层数、隐藏维度、Dropout 或采样次数，观察模型性能与不确定性的变化；
3. **课程作业**：要求学生在现有 Notebook 基础上增加新的数据集、模型或解释方法；

---

## 七、注意事项

1. 若 RDKit 绘图模块在本地环境中不可用，分子可视化部分可能需要额外配置。
2. 若 GPU 环境不可用，代码也可在 CPU 上运行，但训练与解释速度会明显变慢。
3. 不确定性估计和可解释性分析部分依赖前面训练得到的模型，因此建议不要随意删除已保存的模型文件。


---

## 八、第二章代码的整体目标

从整体上看，这三份 Notebook 共同构成了一个面向教学的最小闭环：先完成**分子性质预测**，再讨论模型的**不确定性评估**，最后分析模型的**可解释性证据**。它们不仅展示了图神经网络在药物设计中的基本应用流程，也体现了当前分子 AI 中“预测—置信—解释”一体化分析的基本思路。
